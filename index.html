    How to get a GitHub token

    A Personal Access Token (PAT) is like a scoped password that lets this app commit your README to a repo on your behalf. It works entirely from your browser — there's no server involved. Takes about a minute to set up.

    1Open the token page

    Go to github.com/settings/personal-access-tokens/new (Or on GitHub: your avatar → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token.)

    2Name it & set an expiry
    • Token name: anything, e.g. readme-editor.
    • Expiration: pick whatever you're comfortable with — a short window like 30 days is safest.
    3Choose which repos it can touch

    Under Repository access, choose Only select repositories and pick the repo(s) you'll publish your README to. This limits the token so it can never touch anything else.

    4Give it the one permission it needs

    Under Permissions → Repository permissions, find Contents and set it to Read and write. Leave everything else as “No access”.

    Contents: Read and write is the only permission required — it's what lets the app create or update the README file.
    5Generate & copy

    Click Generate token, then copy it (it starts with github_pat_…). Copy it now — GitHub only shows it once.

    Then, back in this app
    1. Paste the token into the Personal Access Token box.
    2. Optionally tick Remember on this device (skip it on shared computers — it's stored as plain text in this browser).
    3. Click Connect — you should see “Connected as yourname”.
    4. Pick your repository, leave the path as README.md, and click Publish.
    Your token is sent only to api.github.com straight from your browser — never to us (this app has no server). You can hit Forget token any time, and you can revoke it on GitHub under Settings → Developer settings → Fine-grained tokens.

    About GitHub README Editor

    GitHub README Editor is a free, browser-based tool for writing and publishing GitHub README files — with a word-processor-style editor, a live Markdown preview, ready-made templates, badges and widgets, license and community-health file generators, and one-click publishing straight to GitHub.

    It runs entirely in your browser — no server, no accounts, and your files never leave your computer.
